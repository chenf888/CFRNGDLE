# CFRNGDLE

一个 11 位随机数字生成器 + 徽章收集游戏，原型为 [RNGdle](https://www.rngdle.com)。

## 玩法

点击 **Generate** 生成一个 11 位随机数，从右到左逐位闪烁揭示。根据数字的数学性质和模式获得徽章，每个徽章有 TP 分值。

徽章列表按稀有度排序（最稀有关在前），默认显示前 20 个，可展开查看全部。本轮新获得的徽章置顶标注 ✨ 新获得。

### 终极收藏品

部分徽章概率极低（< 10⁻⁷），但数学上可达。11 位数字允许前导 0，因此 `00000000002` 会被识别为 2，**小数字同样可以触发**：

| 徽章 | 可达数量 | 示例 |
|------|---------|------|
| 2 的幂 | 37 个 | 1、2、4、8、16…2³⁶ |
| 3 的幂 | 23 个 | 1、3、9、27、81…3²² |
| 5 的幂 | 16 个 | 1、5、25、125…5¹⁵ |
| 阶乘 | 15 个 | 0!~14!（1、2、6、24…87,178,291,200） |
| 完全数 | 6 个 | 6、28、496、8128、33,550,336、8,589,869,056 |
| 卡塔兰数 | 20 个 | 1、2、5、14、42、132…6,564,120,420 |
| 梅森素数 | 8 个 | 3、7、31、127、8,191…2,147,483,647 |

这些是游戏的"终极收藏品"——存在的意义是让玩家知道"有生之年也许……"。

## 数据持久化

所有已获得徽章和总 TP 自动保存在浏览器 localStorage，刷新不丢失。

## 技术

纯前端，无依赖 — HTML / CSS / 原生 JS 六模块组合。

```
HYPERNGDLE.html   → 入口页面
style.css          → 全部样式
badge-utils.js     → 工具函数库（120+ 函数）
badge-defs.js      → 596 个徽章定义
badges.js          → 徽章管理模块（稀有度排序、分页、localStorage 持久化）
app.js             → 主应用逻辑（11 位生成、逐位揭示）
```
